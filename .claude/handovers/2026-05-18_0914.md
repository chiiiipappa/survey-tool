## 決定事項

- 集計軸の選択状態は API に保存しない（フロントエンドステートのみ）
  - 旧: `saveStep2Axis(token, cols)` を毎チェック時に呼んでいた → 廃止
  - 理由: 軸選択は一時的な操作であり、FA表生成リクエスト時にのみサーバーに送れば十分

- STEP1 ナビボタンを1本に統合（upload と questions を同じボタンで切り替え）
  - sessionToken 有無で遷移先を振り分ける

- CSV 読込時に集計軸のデフォルト選択を自動実行
  - SA/S/NU/N かつ子設問なしの設問を `step1AxisCodes` に自動セット

---

## 捨てた選択肢と理由

- **`saveStep2Axis` の維持**: 軸選択をサーバーに逐次保存する仕組みはセッション管理と相性が悪く、ページリロードでの復元も実装していないため廃止した。
- **STEP1/STEP2 ボタンを別々に維持**: upload と questions は1つのフローの中にあり、ユーザーには「STEP1」として見せるほうが直感的。ボタンを分けると状態管理が複雑になるため統合。

## 学び

- `activatePanel` を export しないと他モジュール（questions.js, step2.js）から呼べない。循環importに注意しながら app.js から export するのが正しい構造。
- `deriveStep1AxisDefaults` は `has_children` フィルタが重要。親設問を集計軸にすると集計が崩れる。
