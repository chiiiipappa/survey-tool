## 捨てた選択肢と理由

- `step3-special-add-btn-area` を各パネル（average/attribute）の中に置く → hidden 状態の要素は `getElementById` で取得できるが、z-index やレイアウト上の問題を避けるため結果エリア付近に配置
- 「何点満点か」を number input にする → ドロップダウン（5/7/10/100）のほうがタッチしやすく誤入力が少ない

## ハマりどころ

- grep が UTF-8 日本語ファイルで空結果を返す問題 → `python3 -c "..."` で回避
- `step3-special-add-btn-area` を average パネル内に置いたあと、attribute/fan モードで機能しないことに気づき、結果エリア前に移動した

## 学び

- step3.js の state 更新フロー：`setStep3AvgTargets()` → `_emit()` → `survey:statechange` イベント → `_onStateChange()` → `_renderSpecialConfigCard()` → `_renderAveragePanel()` → `_renderAverageTargetSettings()` の順に再描画される
- `el._avgInitialized = true` パターンで委譲イベントの二重登録を防止
