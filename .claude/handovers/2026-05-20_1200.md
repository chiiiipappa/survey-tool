## 決定事項

- **`ALLOWED_CHARTS` を完全廃止**: 禁止ではなく推奨度として表現する設計に統一
- **ボタングループ方式**: `<select option style>` がブラウザ間で動作しないため、button 要素でグレー表示を実現
- **`hiddenChoices` の保存はバックエンド変更不要**: `step3QuestionSettings` の一部として既にシリアライズ対象（`.surv` ファイル）
- **グラフタイプ変更時は `_rerenderQuestionFull`**: グラフ＋表の両方を再描画。色・hiddenChoices・ソート・向きはすべて state から再取得するため自動維持
- **「すべて表示」と「初期状態に戻す」は同一動作**: 初期状態 = 全選択肢表示のため
- **scatter 実装**: X=選択肢インデックス, Y=% の「ドットプロット」形式。真の散布図（2変数間の相関）は今回スコープ外

## 捨てた選択肢と理由

| 選択肢 | 理由 |
|--------|------|
| `<select>` に `<option style="color:grey">` で非推奨表示 | Safari がスタイルを無視するため不採用 |
| グラフタイプを modal で選択 | ワンクリックで変更できるボタングループの方がUXが良い |
| 選択肢パネルを floating popover で実装 | 位置計算が複雑。カード本体内のインライン展開の方が実装シンプルかつ安定 |
| 「すべて表示」と「初期状態に戻す」を別動作に | 初期状態 = 全表示なので機能的に同一。区別しても混乱するだけ |
| scatter を transpose 対応にする | X 軸の意味が逆転して直感的でなくなるため非対応のまま |

## 学び

- `<select>` の `<option>` 要素は CSS スタイリングがブラウザ依存。クロスブラウザ対応には button group が正解
- Chart.js scatter の data 形式は `[{x, y}]`。x 軸を categorical に見せるには `ticks.callback` で数値→ラベルに変換し、`stepSize:1` で整数刻みにする
- Chart.js radar の `scales.r` に `beginAtZero:true` が必要（デフォルトでは最小値付近から始まる）
