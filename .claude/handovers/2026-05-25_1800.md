## 決定事項

- **`isExcluded` はセットオブジェクト内フィールドとして実装**: 別途 `excludedSetIds[]` は不要。`questionSets` に内包され保存/ロードが自動化
- **⋯メニューは CSS クラストグル方式**: `hidden` 属性は一部 CSS リセットで上書きされる可能性がある
- **複製は一覧から即コピー**: 編集タブへ遷移しない（シンプルさ優先）
- **除外 ≠ 削除**: `isExcluded: true` はセット定義を保持したまま STEP3 から非表示にする可逆操作

---

## 捨てた選択肢と理由

- **Capture phase での document リスナー**: `stopPropagation()` + bubbling phase の方がシンプルで副作用なし
- **`excludedSetIds[]` を state.js に追加**: セットオブジェクト内フィールドで十分。state 変更を最小化

---

## ハマりどころ

- **⋯メニューが開かない**: `hidden` 属性 + document バブリングリスナーが「⋯クリック→メニュー表示→document が全閉じ」の競合を起こしていた。`e.stopPropagation()` + CSS `.open` クラスで解決
- **`hidden` 属性の落とし穴**: `element.hidden = false` にしても、CSS に `[hidden] { display: none !important }` があると視覚的に変化しない
