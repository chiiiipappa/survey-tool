### チェックボックスのフォーカス制御
- `mousedown.preventDefault()` でブラウザのネイティブチェックトグルを無効化し、`click` で手動トグルに統一
- `change` イベントは使わない（切り離された DOM で発火しないケースがあるため）

## 捨てた選択肢と理由

- **`change` イベントでのトグル**: 仮想スクロールで DOM が再生成されると切り離された DOM で発火し、状態が反映されない
- **`axisSelected` スナップショット**: `buildRowHtml` 内で一度 Set を作ると仮想スクロールの再描画タイミングで古い状態が残る。live 参照に変更

## ハマりどころ

- **テンプレートリテラル構文エラー（`_buildTableHtml`）**: `}).join("") + (showRowTotal ? ... : "") + \`</tr>\`` の形式でリテラルが正しく閉じられていなかった。`node --input-type=module` で検出し、`${}` 埋め込みに修正（2箇所）
- **`edit-choice-filter-list` の選択肢フィルタ**: HTML 構造と CSS が揃わないと正しく表示されないため、三つ（HTML/CSS/JS）の同時確認が必要

## 学び

- 仮想スクロール実装では「stale snapshot」と「フォーカス誘発スクロール」が複合すると、チェックボックス操作が完全に無効化されるように見える
- `mousedown.preventDefault()` は focus 移動を防ぐが、テキスト選択等のデフォルト動作も抑止するため、クラス指定で対象を絞ることが重要
- `_VirtualTable.refresh()` を追加することで外部から強制再描画できる出口を作ると、状態変更後に即座にビューへ反映できる
