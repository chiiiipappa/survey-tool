## 決定事項

- **splitDatasetIndices 方式を採用**: 各ページが「どの仮想データセット index を表示するか」を配列で管理。非表示 = どのページにも含まれない index
- **reflowSplitPages の挿入位置**: グループページ群の先頭位置に新ページを挿入（非連続ページのケースは許容）
- **空ページの自動削除**: `moveSplitGraphToAdjacentPage` で移動後にページ内 index が空になった場合は自動削除
- **後方互換維持**: 旧 `splitChunkStart/End` 形式は JS・Python 双方で fallback として残す（既存プロジェクトファイルを壊さない）
- **グラフ一覧は全グラフを表示**: アクティブページ限定ではなく、同一 chartResultId の全グループページ分を一覧化

## 捨てた選択肢と理由

- **`splitChunkStart/End` の拡張案**: 連続スライスのままページ内の選択肢制御を追加する案 → 非連続配置（グラフの移動）が不可能なため廃止
- **ページ単位でのグラフ一覧分離**: 「このページのグラフだけ表示」案 → ページ間移動の全体像が見えないため、全グループ表示に統一
- **`splitHiddenIndices` フィールドの追加案**: 非表示情報を別フィールドで管理する案 → インデックス配列に含めないことで「非表示＝未割当」と表現でき、データモデルをシンプルに保てる

## ハマりどころ

- `_pid` ヘルパーが state.js の line 1162 で `const` 定義されているが、新関数群は line ~890 付近に挿入したため「前方参照」になっている。JavaScript では `const` は TDZ（一時的デッドゾーン）だが、クロージャが「呼び出されるとき」にルックアップするため、モジュールロード完了後に呼ばれる関数からは問題なくアクセスできる（動作確認済み）
- `reflowSplitPages` の `nonGroup.slice(firstIdx)` ロジック: `firstIdx` は `allPages` 内のインデックスだが `nonGroup` は別配列。非連続グループページが挟まっている場合に位置がズレるが、実用上（グループページは連続追加される）は正しく動作する
