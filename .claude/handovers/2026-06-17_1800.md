## 決定事項

- **axisCode="" = 全体集計**: バックエンドの既存「軸なしモード」を活用。axis_categories=["全体"]、N=全体n数を返す
- **overrideCodes=[] = 全設問**: バックエンドは `target_question_codes=[]` のとき全設問を集計対象にする（既存仕様）
- **深掘りモード・集計セットなし**: 比較軸あり → 全設問×比較軸（有用）、比較軸なし → 全設問全体集計
- **`_runCrosstab()` の引数なし呼び出し（L2542）**: overrideCodes が undefined → else パスでセット参照 → 既存動作維持

## 捨てた選択肢と理由

- **selectedCodes 空のまま `_runCrosstab(null, [])` を渡す（既存のまま）**
  - 却下: `if (overrideCodes)` で `[]` が truthy になり targetCodes=[] → 即エラー。そのまま渡せない
- **深掘りモードで basicAxisCode を target として集計する**
  - 却下: basicAxis は filter column として使用。target にすると「ロート製薬の中でのロート製薬分布」= 100% になり無意味
- **overrideCodes の型を変えてセンチネル値を導入する（例: Symbol('all')）**
  - 却下: 既存の null/undefined 規約と矛盾。`overrideCodes !== null && overrideCodes !== undefined` で十分明示的

## ハマりどころ

- **JS の空配列は truthy**: `if ([])` → true。`if (overrideCodes)` で `overrideCodes=[]` を通過させようとするとハマる
  - 対策: `overrideCodes !== null && overrideCodes !== undefined` でチェックを明示化
- **axisCode="" は `!axisCode` が true**: もし `if (!axisCode || ...)` のままにすると全体集計モードに入れない
  - 対策: `axisCode === null || axisCode === undefined` に変更

## 学び

- バックエンド `step3.py` の 3 つのモード分岐:
  - `if is_bracket_ma_axis:` → bracket MA型軸モード
  - `if not axis_col_raw:` → 全体集計モード（軸なし）
  - 通常軸モード
- `Step3CrosstabRequest.axis_question_code: str` は `""` を受け付ける（Pydantic は None と空文字を区別する）
- `overrideCodes=[]` → `target_question_codes=[]` → バックエンドが全設問を集計対象にする（既存動作の副作用を有効活用）
