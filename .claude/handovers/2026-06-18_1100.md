## 決定事項

- **アプリ形式の出力ファイル名**: `survey-tool-mac.dmg` / `survey-tool-windows.exe`（標準化された名前。electron-builder が生成するバージョン入りのファイル名とは別）
- **build-release.sh は呼ばない**: `build_app` は electron-builder を直接実行し、`build-release.sh` はバイパスする。理由：sh はインストーラをZIPで包む処理をするため、dmg/exe 単体コピーと相性が悪い
- **release/README.txt は動的生成**: 生成したアーティファクトの種類に基づいて内容が変わる。ZIPのみなら起動スクリプト手順、アプリのみならインストーラ手順、両方なら両方を記載

## 捨てた選択肢と理由

- **build-release.sh 経由でアプリ形式を生成する案**: sh がインストーラをZIPで包んで出力するため、release/ に dmg/exe 単体を置けない。バイパスしてPythonから直接 electron-builder を呼ぶ方がシンプル
- **release/README.txt を手書きで維持する案**: 生成形式ごとに README を手動で更新するのは忘れやすい。動的生成に統一した

## 学び

- electron-builder は Mac 環境から `-mw`（Mac + Windows同時ビルド）で Windows exe も生成できる（Wine 不要）
- `zipfile.ZipInfo.external_attr` に `(mode << 16)` で Unix パーミッションを埋め込める（`.command` ファイルに実行権限付与）
- `dist/*.exe` のglobには `.blockmap` ファイル（`*.exe.blockmap`）も引っかかるため、`.blockmap` 拡張子除外が必要
