## 決定事項

- **キャッシュキー形式**: `"${axisCode}||${secAxisCode||""}||${setId}"` — `||` で区切ることで軸コードに含まれる可能性がある `_` と衝突しない
- **クロス集計結果の保存**: メモリキャッシュに加え `.surv` ファイルにも `step3_crosstab_cache` として保存（ロード後に即時表示可能）
- **`_lastCrosstabData` の後方互換**: `getLastCrosstabData()` を使う step3_export.js との互換性のため、`_runCrosstab()` と生成済みセットナビのクリック時に常に `_lastCrosstabData` を同期
- **自動推定ルール**: `question_code.match(/^([A-Za-z]+\d+)/)` のプレフィックスでグループ化。先頭に「全設問（FA除く）」を追加
- **FA タイプの扱い**: FA/OA/OE/FT/FN は設問セットのグループ化には含めるが、「全設問（FA除く）」から除外

---

## 捨てた選択肢と理由

| 選択肢 | 理由 |
|--------|------|
| `_lastCrosstabData` を廃止して `_currentCacheKey` に統一 | step3_export.js が `getLastCrosstabData()` に依存しており、修正範囲が広がる。後方互換のため同期維持が最小変更 |
| クロス集計結果をプロジェクトファイルに保存しない | ユーザーが「保存する」を選択したため採用しなかった |
| キャッシュキーを `${axisCode}__${setId}` にする | `__` は既存コードで様々な用途に使われているため衝突リスクあり。`||` を使用 |
| STEP2 に設問セット管理 UI を追加 | STEP1 の方が設問の文脈に合っており、STEP2 は主に回答データの確認画面のため |

## 学び

- `IntersectionObserver` の `rootMargin: "200px"` は「スクロールして表示される200px前」に描画を開始するため、ユーザーには一括描画されているように見える
- step3.js 内の `_lastCrosstabData` は多数の箇所で参照されているため廃止より同期維持が現実的
- `question_sets` は **フロントエンドのみで管理** できる（バックエンドは `target_question_codes` パラメータを既にサポートしていたため API 変更不要）
