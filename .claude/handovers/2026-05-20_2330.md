## 決定事項

### STEP 役割分担の再整理
| ステップ | 担当 |
|----------|------|
| STEP1 | 調査票確認・集計軸候補設定・分析セット設計 |
| STEP2 | 回答データ読込・FA表形成 |
| STEP3 | 集計軸選択→カラー確認→セット実行→グラフ出力 |

### STEP3 の分析セット操作方針
- 「セットを選ぶ」概念は STEP3 に不要
- STEP1 で設計したセットを STEP3 で実行するだけ
- 各セットに独立した「生成」ボタン（複数セットの並行生成が可能）

### STEP3 カラー設定の配置
- 集計軸を選んだ直後にカラーを確認・変更できる流れが自然
- STEP1 に置くと「分析設計」の流れと噛み合わない

---

## 捨てた選択肢と理由

### STEP3 に「active set」概念を残す案
- 「1セットを選択状態にして一括生成」方式も検討
- 却下理由：「選択」概念そのものを STEP3 から排除したいため。per-set ボタンで十分

### JS を step3.js に移動してカラーセクションを再実装する案
- カラー設定 HTML の ID を変更して step3.js で管理する案
- 却下理由：`updateAxisColorSection()` が既に DOM ID 参照であり、HTML の位置移動だけで動作する。不要な実装コスト

## 学び

- `updateAxisColorSection()` は `survey:statechange` 経由で全パネル共通に呼ばれるため、HTML の物理的な位置に依存しない設計になっている
- STEP3 の `_runCrosstab()` は `setId` を引数で受け取れるようになったことで、外部から直接セットIDを指定して集計実行できる（将来の一括生成等にも使いやすい）
