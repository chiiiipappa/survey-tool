## 決定事項

### chartMode → PowerPoint チャート型マッピング
| chartMode | PowerPoint 型 | 理由 |
|-----------|-------------|------|
| vbar / grouped_vbar / auto / brand_hbar | COLUMN_CLUSTERED | brand_hbar は indexAxis="x"（縦棒）|
| hbar / grouped_hbar / brand_vbar | BAR_CLUSTERED | brand_vbar は indexAxis="y"（横棒）|
| stacked100_vbar / brand_vbar_stacked | COLUMN_STACKED_100 | |
| stacked100_hbar | BAR_STACKED_100 | |
| small_multiples | COLUMN_CLUSTERED × N | スライド内に横並び |

- `brand_hbar` の命名は「チャート全体の横幅（ブランド名を x 軸に並べる）」を指すため、実体は縦棒（COLUMN）
- データラベルは % フォーマット（`0.0%` 等）を numFmt XML で設定。percents は 0–100 の値で受け取り、PPTX チャートへは 0.0–1.0 に変換して渡す

### スライドレイアウト
- 16:9 (13.33" × 7.5")
- タイトル 0.47"、サブタイトル 0.36"、コンテンツ上端 1.0"
- showTable=false: グラフ 6.25"
- showTable=true: グラフ 58% / 表 40%（gap 0.07"）
- table_only: 表 6.25"

### 集計表
- 全セル Noto Sans JP
- 先頭列幅: 22%、残り均等分
- tableContentMode: percent / count / both (改行区切り)
- 集計表の合計列は `showTableColTotal=true` or 軸カテゴリが複数のとき表示

## 捨てた選択肢と理由

- **画像変換（chart → PNG → スライド挿入）**: 要件で明示的に禁止。PowerPoint でグラフが編集不可になるため
- **slide_layout[0]（タイトルスライドレイアウト）**: プレースホルダーが干渉するため、blank_layout (index=6) を使用
- **`table.cell.fill` で背景色設定**: ヘッダ行に色を付けると文字が読みにくくなる懸念があり、今回は実装せず（将来対応）
- **ページごとに複数スライド（small_multiples）**: STEP4 の見た目再現を優先し、1 ページ = 1 スライドで小分割グラフを横並びに収める方針を選択

## ハマりどころ

- **percents の単位**: STEP4 の percents は 0–100 の値。python-pptx の ChartData に渡すのは 0.0–1.0 が一般的（PowerPoint がパーセント表示する）。`/ 100.0` で変換している
- **データラベルの % フォーマット**: `plot.has_data_labels = True` だけでは `0.6` のまま表示される。`c:numFmt` XML 要素を手動で追加して `formatCode="0.0%"` を設定する必要がある
- **テーブル罫線**: python-pptx の高レベル API では罫線設定に限界があるため、lxml で `a:lnL/lnR/lnT/lnB` 要素を直接追加する
- **ポート衝突**: 既存サーバーが 8002 で動作中だったため、テスト用に 8003 を使用

## 学び

- `python-pptx` の ChartData は `add_series(name, values)` で系列を追加。values は float のリスト
- 系列カラー設定: `series.format.fill.solid()` → `series.format.fill.fore_color.rgb = RGBColor.from_string(hex)`
- `XL_CHART_TYPE.COLUMN_STACKED_100` と `BAR_STACKED_100` は別エントリ（stacked も別）
- python-pptx 1.0.x は `XlsxWriter` と `Pillow` も依存関係として一緒にインストールされる
