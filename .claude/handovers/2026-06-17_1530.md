## 決定事項

- 平均点指標は「通常分析で使う**指標**」（平均点指標セクション）として残しつつ、**同時に「基本軸・比較軸」にも追加**する設計
  - 理由: ファン度（DERIVED型）と同様に基本軸でも使えると便利。スコア値ごとのクロス集計（例：NPS 0〜10点×設問）が有効なユースケース
- 数値軸は整数なら "5"、小数なら "5.5" 形式で文字列化（`int(v) == v` で判定）

## 捨てた選択肢と理由

- **SCORE型アイテムを step2AxisCandidates に追加しない**（現状維持）
  - 却下理由: ユーザーが「基本軸・比較軸に出てこない」と報告しており、ファン度と同様に出てくるべき
- **平均点指標のみに絞る**（DERIVED optgroupに入れない）
  - 却下理由: ★ 特定分析で作成した軸・指標グループに含めた方がユーザーが見つけやすい
- **Playwright でブラウザテスト**
  - 試みたが `file-input` IDが `#upload-layout-input` でなく `#file-input` だったり、セッショントークンの取得が難しく断念
  - API直接テストで代替（十分に検証できた）

## ハマりどころ

- **サーバーが `--reload` なしで起動**していたため、ファイル変更が反映されず「修正したはずなのに動かない」状態が続いた
  - `ps aux | grep uvicorn` で確認して判明。`axis_categories: ['5.0', ...]` が "5.0" のままだったのが手がかり
- **float列を string と比較**すると pandas は False を返す（TypeError でなく静かに失敗）
  - `df["col"] == "5.0"` where col is float64 → all False → totals all 0

## 学び

- `pd.api.types.is_numeric_dtype` は float64 に対してTrueを返す（確認済み）
- `int(v) == v` はfloat vが整数値かどうか判定するPythonイディオム（5.0 == 5 → True）
- uvicornの `--reload-dir` は `--reload` なしでは機能しない
