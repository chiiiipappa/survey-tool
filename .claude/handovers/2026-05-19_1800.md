## 決定事項

- **フロントが色を解決してバックエンドに送る**: `resolved_colors` に `_getColorsForGraph` で解決した hex 色配列を渡す。バックエンドに色解決ロジックを複製しない
- **Excel 一括 = 1ファイル複数シート**: 使いやすさ最優先（CSV 一括は ZIP）
- **PNG はクライアントサイドのみ**: `canvas.toDataURL` で完結、サーバー不要
- **`hiddenChoices` / `graphTitle` は `step3_question_settings` に自然に含まれる**: バックエンドスキーマ変更不要で `.surv` 保存に自動的に含まれる
- **`openpyxl` 既存インストール活用**: `xlsxwriter` 追加インストール不要

## 捨てた選択肢と理由

| 選択肢 | 理由 |
|--------|------|
| `xlsxwriter` を使う | `openpyxl` が既インストール済みで機能的に十分 |
| バックエンドで色解決する | フロントに色解決ロジックがすでにあり（固定パレット・カスタムカラー等の複雑な優先順位）、複製するとバグの温床になる |
| PNG を ZIP にまとめる | JSZip が未導入でサーバー側では Chart.js レンダリング不可。個別ダウンロードで十分 |
| CSV 全設問を1ファイルにまとめる | 設問ごとに独立した CSV の方が後処理しやすい |

## ハマりどころ

- `step3.js` は ES module だが、今まで `export` は `initStep3Panel` のみだった。内部 `_xxx` 関数を `export { _fn as fn }` 形式で公開する必要があった（`_lastCrosstabData` / `_charts` は getter 関数経由で公開）
- openpyxl の `DataPoint` インポートパス: `from openpyxl.chart.series import DataPoint`（旧バージョンでは違う場所にある場合あり）
- Circular import に注意: `step3_export.js` → `step3.js` のインポートは問題ないが、逆は不可

## 学び

- openpyxl の系列色: `series.graphicalProperties.solidFill = "RRGGBB"` で直接指定（`#` は不要）
- openpyxl の pie chart DataPoint: `chart.series[0].dPt.append(dp)` でアイテムを追加する
- `params.toString()` / `params.size` で URLSearchParams の空チェックができる（`params.size` は比較的新しいAPI）
