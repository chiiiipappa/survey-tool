## 決定事項

- `modal-overlay` CSS は汎用クラスとして定義（`.modal-overlay` + `.modal-dialog` + `.modal-header/body/footer`）、既存の `step3-duplicate-modal` も同 CSS を使用するので `btn-ghost` → `btn-secondary` に統一
- 新規ボタンは「常に確認ダイアログを表示」（isDirty に関わらず）
- `_build_axis_cats`: STEP1 選択肢定義がある場合は全件返す（0件選択肢も含む）。STEP1 定義がない場合のみデータから生成（従来動作）
- 合計列はデータ合計の算出方法が aggMode によって異なる（count/col_pct/row_pct/composition すべてで合計を計算）
- 異常値チェック（赤文字）は %表のみ適用（N表はなし）。aggMode に関わらず 99.5〜100.5% 範囲外で発火

## 捨てた選択肢と理由

- **新規ボタン: ネイティブ `confirm()` を継続使用** → カスタムボタンラベル（「新規作成する」）が実現できないためカスタムモーダルに変更
- **循環インポート解消のためのカスタムイベント方式** → `app.js → questions.js → report.js → app.js` のサイクルは ES モジュールのライブバインディングで既存から動いていたため不要と判断
- **STEP3: ラベル変換失敗時にデータ値の除外** → 除外すると N 数が合わなくなる。代わりにデータ値を末尾に「余剰列」として追加し、変換失敗を視覚化する方針を採用
- **PPTX 合計列: 従来の末尾配置を維持** → ユーザー要求（「集計軸の直後に」）に従い第2列に変更

## ハマりどころ

- **`.modal-overlay` CSS が未定義**：既存の `step3-duplicate-modal` も `modal-overlay` クラスだが CSS なし。`show_col_total` → `modal.style.display = ""` で表示したが視覚的に見えないため「ボタン反応なし」と誤認された
- **`btn-ghost` クラス未定義**：`app.css` に存在しないクラスを使用していた。`btn-secondary` に変更
- **`step2.js` の `_rawRows` / `_labeledRows` 宣言位置**：`_resetStep2UI` から参照する際、宣言が下にあるが `var` ではなく `let` のため問題なし（関数宣言のホイストはあるが変数はない）

## 学び

- `step3.js` の `_buildPctTable` は aggMode ごとに `cellVal` の計算が異なるため、合計値の計算も同じ `cellVal` を呼び出すことで一貫性を保てる（直接 `percents` を参照しないことが重要）
- PPTX の `_add_table_shape` は `lc`（LayoutConfig：ページレイアウト設定）と `cc`（ChartConfig：グラフ設定）を別々に受け取る。表示設定は `cc` 側から取るべき（質問個別設定）
