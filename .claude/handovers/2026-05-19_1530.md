## 決定事項

- **年代別A/B**: A（青系）が自動検出デフォルト。B（緑系）は手動選択のみ（ラベルが同じため自動判別不可）
- **固定カラーの優先順位**: `customColors` > 固定カラー（その他/全体） > 固定パレット > COLORS デフォルト
- **`step1AxisColors` の形式**: `{ [axisCode]: { fixedPalette: string | null } }`
  - `null` = "なし（デフォルト配色）"
  - エントリ自体が存在しない = 自動検出
  - 旧 `{ palette: "warm" }` は自動検出フォールバック（互換性あり）
- **D&Dの保存**: 系列順を変えた場合も `customColors` 配列として保存（順序が反映される）
  - `seriesOrder` などの専用フィールドは不要と判断

## 捨てた選択肢と理由

| 選択肢 | 理由 |
|--------|------|
| ソフトパレット（6種）を残しつつ固定パレットを追加 | UI が複雑になる。既存データで旧パレット名が保存されていても自動検出フォールバックで対応できるため廃止 |
| `_colorModalLabels` をモジュール変数として保持 | `_colorModalIdx` があれば `_getColorSeriesLabels` で再導出可能。二重管理は不要 |
| 行ドラッグに `seriesOrder` 専用フィールドを追加 | `customColors` 配列の順序に反映するだけで足りる。スキーマ変更・backend対応が不要 |

## 学び

- HTML5 Drag & Drop: `dragover` に `e.preventDefault()` が必要（ないと `drop` が発火しない）
- `dragleave` でエリア外に出た判定: `!container.contains(e.relatedTarget)` で確実に判別できる
- パレット定数（`FIXED_PALETTES`, `FIXED_PALETTE_ORDER`）は `step3.js` と `questions.js` の両方に定義が必要（ES module で共有する共通ファイルがないため）
