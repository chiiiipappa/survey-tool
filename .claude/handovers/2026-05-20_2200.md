## 決定事項

- **`age_c` を年代系自動検出のデフォルトパレット**として採用（旧 `age_a`）
- **`canonicalValues`** を FIXED_PALETTES の各エントリに追加して「パレット定義 = 値↔色リスト」を明示
- **`valueColorMapping`** を `step3QuestionSettings[qCode]` に保存（Python スキーマ変更不要）
- **ファジーマッチング優先順位**: 完全一致 > 年代正規化一致 > 部分一致（label同士の包含関係）
- **`overriddenSeriesColors`** は後方互換のため維持（実ラベルへの直接上書き）
- **`__none__` を未検出時のデフォルト返り値**とし、グレー単色が自動適用される設計に統一
- **STEP3 連携は `_getColorsForGraph` の修正のみ**で実現（グラフ・PNG・Excel エクスポートすべてに自動反映）

---

## 捨てた選択肢と理由

| 選択肢 | 理由 |
|--------|------|
| `valueColorMapping` を STEP1 軸レベルで管理 | STEP3 モーダルの操作単位（設問単位）と合わず、複雑になる。STEP3 設問設定に持たせる方が現行アーキテクチャと整合する |
| ドラッグ並び替え（value mapping row） | 実装コストに対して使用頻度が低い。追加・削除で代替可能 |
| `valueColorMapping` を `step1AxisColors` にも持つ | state の二重管理になりリスクが高い。`step3QuestionSettings` 側のみで統一 |
| パレット選択 → 即時グラフ適用（モーダル不要） | 既存の「このグラフだけ / 全グラフ」承認フローを壊さないよう維持 |
| Python スキーマに新フィールドを追加 | `step3_question_settings` が `dict` 型なので透過的に保存・復元可能 |

## 学び

- `FIXED_PALETTES` の `colorFor()` は旧来 `(\d+)代(男性|女性)` の固定順のみだったため、「男性20代」形式の実データに未マッチだった。性別と年代を別々の regex で取る設計に変更するだけで両順・範囲表記に対応できる
- `_getColorsForGraph` が単一の色解決エントリポイントであるため、`valueColorMapping` レイヤーを1か所追加するだけでグラフ・PNG・Excel の全出力に自動反映される
- `_detectFixedPaletteFromQuestion` は questions.js の STEP1 UI 用、`_detectFixedPaletteFromLabels` は step3.js の STEP3 ランタイム用で役割が分かれている。両方を同じ方向に揃える必要があった
