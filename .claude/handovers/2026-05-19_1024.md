## 決定事項

- **ファンラベルフィルタは全角・半角両対応**: `/[*＊]ファンラベル/` で統一。データソースが全角のみでも将来の変更に対応
- **「追加読み込み」ボタンは削除**: step2-loaded-card の旧ボタンのうち、ユーザー要件（再読込・差し替え・読込解除）に含まれないため削除
- **再読込はファイルオブジェクト保持方式**: API にはファイルをアップロードし直す仕様。セッション内に限り `_lastFile` に File オブジェクトを保持して再アップロード
- **`_resetStep2` はフロントのみリセット**: AppState の step2 フィールドはリセットしない（画面非表示のみ）。必要であれば `setStep2UploadResult(null)` 系の処理を追加検討

---

## 捨てた選択肢と理由

- **「検索・フィルタ」をカードタイトルとして残す案**: ユーザーが「1つの枠にしたい」と明示したため不採用
- **drop-zone-loaded スタイルを維持してD&D追加**: hover が無効化されD&Dのフィードバックが出ないため、通常の drop-zone に戻した
- **`_resetStep2` で AppState もリセット**: step2 フィールドのリセット関数が存在せず、STEP1 全体リセットになる恐れがあるため今回は画面表示のみに絞った

## 学び

- `questions.js` の `btn-csv-replace` では `replaceDropZone.addEventListener("click", ...)` と `btn-csv-replace` ボタンの両方にクリックハンドラを付けたため、ドロップゾーンのクリックとボタンクリックが二重に反応しないか注意（`replaceDropZone` はカード全体ではなく div に限定されているため問題なし）
- step2 の既存 `_initDropZone` では `classList.add("drag-over")` （ハイフンあり）を使っているが、CSS は `.drop-zone.dragover`（ハイフンなし）。既存バグだが今回の新規 D&Dゾーンは `"dragover"` で統一
