## 決定事項

- **`question_type` 11カテゴリ**: SA / MA / MATRIX / NUMERIC / OA_TEXT / OA_AUX / WEIGHT / ATTRIBUTE / FLAG / DERIVED / UNKNOWN
- **分類優先順**: WEIGHT > FLAG > ATTRIBUTE > DERIVED > OA_TEXT（type_code優先、次にキーワード）> MATRIX > MA > SA > NUMERIC > UNKNOWN
- **XL type_code → OA_TEXT**: `Q3X18_1`(type=XL)→OA_TEXT という要件に基づく。X18 の番号だけでは判定しない
- **OA_AUX**: 親が OA_TEXT の子設問を 2 パス目で自動判定
- **分析対象 OFF のデフォルト**: OA_AUX / FLAG / DERIVED の 3 カテゴリのみ（OA_TEXT はデフォル ON）
- **OA_TEXT の扱い**: 分析対象 ON だが `_SKIP_TYPES` により通常クロス集計からは除外。カスタムセットには所属可能（FA分析用）
- **システムセット（自動）**: `STEP3_DEFAULT_TYPES`（SA/MA/MATRIX/NUMERIC/WEIGHT/ATTRIBUTE/UNKNOWN）＋除外設問を除いた設問のみ。OA_TEXT はシステムセットに含まれない
- **カスタムセット作成モーダル**: OA_TEXT を含むが「自由回答（FA分析用）」セクションに分離表示
- **`excludedQuestionCodes` 保存キー**: `excluded_questions`（LayoutSaveData）
- **`hidden_question_types` バグ修正**: `ProjectSaveRequest` にフィールドが欠けていたため runtime error になっていた（修正済み）

## 捨てた選択肢と理由

- **X18/X19 番号でOA判定**: ユーザー要件で明示的に禁止。type_code ベースの判定のみ使用
- **OA_TEXT をデフォルト除外**: FA分析で有用なため ON にした。分析対象は user が手動で OFF にできる
- **「分析対象」チェック変更時にシステムセットを再生成**: 処理が重いため、今回は次回ページロード/再検出時に反映される設計
- **MATRIX を `is_child=True` の SA 子設問で検出**: ML/SL type_code 判定のみに限定。子設問の型だけでは誤判定リスクがある
- **`manual_override_type` の UI**: schema フィールドは追加したが、オーバーライド UI は今セッションでは未実装（将来対応）

## 学び

- classify_question_type() の 2 パス設計（1パス目: 個別判定、2パス目: OA_AUX判定）が効果的。親の question_type が確定してから子を判定できる
