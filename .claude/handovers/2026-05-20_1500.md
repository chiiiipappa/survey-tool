## 決定事項

- **複合ラベルの形式**: `"axis2値 × axis1値"`（MINI × コアファン）。axis2 が外ループ（ブランド）、axis1 が内ループ（ファンラベル）
- **バックエンドはカルテジアン積のみ生成**: N数フィルターはフロントエンド側で適用（閾値をインタラクティブに変更できるため）
- **split モード変更時は全体再描画** (`_rerenderCompositeAll`): 個別グラフ ID が変わるため、部分更新より全体再描画を選択
- **`_compositeColorPaletteLookup` パターン**: `_getColorsForGraph` を変更せず module-level 変数で一時的にパレット検索ラベルを差し替える方式。Chart.js ビルダーへの引数変更なしで実現
- **ネスト表示 = フラット表示と同一実装**: Chart.js での真のネスト軸ラベルにはプラグインが必要なため、今回は「同じ Chart でラベル順が視覚的グループを形成する」として代替
- **export（Excel/CSV）**: 既存ロジックがそのまま動作するため専用対応は今回スコープ外

## 捨てた選択肢と理由

| 選択肢 | 理由 |
|--------|------|
| `_renderChartInArea` に `forcedColors` 引数追加 | 全チャートビルダー関数の変更が必要になり変更量が大きい |
| `overriddenSeriesColors` に一時色を書き込む | AppState の実際の色設定を汚染する |
| split 時に個別グラフのみ再描画 | グラフ ID が可変（`${idx}-${gIdx}` 形式）で再描画対象特定が複雑 |
| `_renderAxisSelector` を index.html に静的 HTML で追加 | axis② の候補リストは動的（STEP1 選択に依存）なので JS で生成する必要あり |

## 学び

- pandas の複合列生成は `df["__composite__"] = df[col1] + sep + df[col2]` で単純に作れる。既存の `_crosstab_sa` / `_crosstab_ma` をそのまま流用できる
- Chart.js のカラー解決はビルダー内で `_getColorsForGraph` を呼んでいるため、module-level 変数で一時的にオーバーライドするパターンが最小変更で済む
- N数フィルターはレンダリング時にインデックス配列 `keepIndices` を作って counts/percents を両方スライスする。バックエンド側での除外は不要
