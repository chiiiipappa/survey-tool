## 決定事項

- **16:9 は CSS `aspect-ratio: 16/9` で実現**。JS で高さを明示した場合は CSS を上書き（ブラウザ仕様通り）。
- **pie チャートは `height: auto`** で aspect-ratio を意図的に無視（複数パイが横並びになる特性上、固定比率が合わない）
- **高さスライダーのデフォルト表示値は 270**（スライダー UI 上での中間値として。実際の高さは CSS aspect-ratio が決定する）
- **棒の太さは `barPercentage`**（Chart.js 4.x の dataset-level オプション）。`barThickness`（固定px）ではなく相対指定
- `barPercentage` のデフォルトは **0.9**（Chart.js のデフォルト値。`null` のときは 0.9 を使う）

## 捨てた選択肢と理由

- **JS で `maintainAspectRatio: true, aspectRatio: 16/9` を使う方法**：CSS の `aspect-ratio` で十分。Chart.js 側で設定すると既存の `maintainAspectRatio: false` 設定（コンテナいっぱいに広がる）を変更する必要があり影響範囲が大きい
- **`barThickness`（固定px）**：カテゴリ数や画面幅によって見た目が大きく変わるため不採用。`barPercentage`（カテゴリ幅の比率）の方が汎用的
- **高さスライダーの `input` と `change` 両方で state 保存**：`input` はライブプレビュー（DOM 直接操作）のみ、`change` で保存とした。両方で保存すると state 更新回数が多くなる

## ハマりどころ

- **棒の太さスライダーの一括バーへの `input` イベント**：当初 `_onResultsChange` に書いたが、`step3-bulk-bar-width` は `change` ではなく `input` を拾う必要があった → `_onResultsInput` に移動
- **Playwright で F1系（集計軸と同じ設問）を選ぶと Internal Server Error**：F1 が axis でもあり target でもある構成は既存バグ。Q1系 など非軸設問を選ぶと正常動作
- **Playwright の `set_input_files` で strict mode violation**：`input[type="file"]` が 6 要素存在するため `#file-input` など ID で指定する必要があった

## 学び

- CSS `aspect-ratio` は JS で `style.height` を直接設定すると上書きされる。明示的な高さなし = CSS aspect-ratio が有効、というシンプルな実装パターンが使える
- Chart.js の `barPercentage` は dataset レベルで設定できる（グローバル options ではなく各 dataset に付与）
- Playwright で `input` イベントを `dispatch_event('input')` でトリガーすると `_onResultsInput` が正常に動作することを確認
