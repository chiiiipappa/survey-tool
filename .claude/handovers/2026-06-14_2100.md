## 決定事項

- **STEP2 `step2-advanced-card`**: 旧カードをそのまま `step2-match-result-card` の `<details>` 内部に格納。div に ID だけ残し display 制御はすべて除去。外側のカードで visibility を制御。
- **`<details>` + `style.display="none"` 共存**: `details` 要素でも `style.display = "none"` / `""` が同一に動作することを確認済み（STEP1 でも採用）
- **`_initCollapsible()` (step2.js)**: `step2-advanced-toggle` / `step2-adv-chevron` を参照する死コード。`initStep2Panel()` から呼ばれていないため放置（削除せず）

## 捨てた選択肢と理由

- **`step2-advanced-card` ごと削除し要素 ID を変更**: JS 全体への影響が大きく (`renderPreviewCard`, `_initTabSwitcher` 等)、ID をそのまま維持して内包する方針に
- **コンパクト表示を別ファイルに分離**: 変更が小さいので `renderMatchCard()` 末尾に追記で完結

## ハマりどころ

- `app.css` の Edit が `File has not been read yet` で失敗 → `Read` 後に再実行で解消
- `_resetStep2()` に `step2-advanced-card` の非表示が残っていると、再ロード後にプレビューが見えなくなる問題 → 削除して外側カード `step2-match-result-card` のみで制御

## 学び

- `<details>` の折りたたみは `style.display="none"` / `""` で完全に JS 制御でき、div と差はない
- 既存 HTML 要素を新カードに「内包する」リファクタは、ID さえ維持すれば JS 変更を最小化できる
